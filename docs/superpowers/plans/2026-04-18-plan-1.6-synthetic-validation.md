# Plan 1.6 — Synthetic-Submission Validation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Prove the full submission → review → decision → site pipeline works end-to-end on 5 curated synthetic papers, before any real user touches the alpha.

**Architecture:** New artifacts under `editor-skill/`: (a) a `test/synthetic/` directory with fixture paper bodies, a policy fixture, an `expected-outcomes.yml` ground-truth file, and a Vitest integration test that runs `runTick` multiple times + a real `astro build` against the fixture; (b) three new CLI subcommands (`prepare-synthetic-fixture`, `simulate-review`, `synthetic-report`) used by a new `/synthetic-validation` slash command for operator-invoked judgment testing with real Claude subagents.

**Tech Stack:** Reuses the existing Node 20 + TypeScript + Vitest + simple-git + js-yaml stack in `editor-skill/`. Integration test spawns a real Astro build (uses `site/` project's node_modules via symlink, same technique as `site/test/build.test.ts`). No new dependencies.

**Scope boundary:** IN — 5 synthetic papers, plumbing test, judgment slash command, 3 new CLI subcommands. OUT — plagiarism test case (no room at N=5), real Worker MCP roundtrip, real Stripe flow, reserve-reviewer fallback path (each paper has 3 external reviewers seeded), multi-round revision flows.

**Spec reference:** `docs/superpowers/specs/2026-04-18-plan-1.6-synthetic-validation-design.md`.

**Cost ceiling reminder:** Plumbing test has zero LLM calls (stubbed subagents). Judgment slash command is manually invoked by the operator and burns Claude Code subscription quota per run.

```markdown
# Electoral Institutions and Legislative Gridlock

**Author:** agent-author01

## Abstract

We use panel data from 60 democracies (1990-2020) to estimate the effect of
electoral-system proportionality on legislative gridlock, measured by delay
to budget passage. We find a statistically significant reduction in gridlock
under more proportional systems, robust to country fixed effects and
instrument choice.

## 1. Introduction

Consider a democracy facing rising fiscal pressure. When a shock arrives,
how quickly can the legislature respond? Prior work (Tsebelis 2002) identifies
veto players as the decisive constraint; we extend this framework by
operationalizing "veto player prevalence" through electoral-system
proportionality. More proportional systems produce more parties, which
increases the number of veto players — but it also dilutes ideological
extremity within coalitions, reducing the coordination cost of passing
compromise budgets.

## 2. Data

We construct a panel of 60 established democracies from 1990 through 2020.
Our outcome is the number of days between the executive's budget proposal
and legislative passage. Our key independent variable is Gallagher's
disproportionality index, coded from election-level seat allocations. We
include country and year fixed effects plus standard controls (GDP growth,
political polarization, divided government indicator).

## 3. Identification

Naive estimates are confounded by endogenous electoral-system choice. We
instrument proportionality using the size of the legislature (which is
largely determined by population and sets mechanical upper bounds on
disproportionality). First-stage F-statistic is 48.2.

## 4. Results

The IV estimate suggests a one-unit increase in Gallagher's index (more
disproportional) increases budget delay by 6.3 days (SE 1.8). Placebo tests
on pre-reform years and heterogeneous-treatment-effect analyses support
the identification strategy.

## 5. Discussion

Our findings complicate the standard veto-player narrative. They also have
normative implications: proportional systems may trade off representation
quality against gridlock cost in ways that depend on the legislature's
institutional capacity.

## References

- Tsebelis, G. (2002). Veto Players: How Political Institutions Work.
- Gallagher, M. (1991). Proportionality, Disproportionality and Electoral Systems.
- Lijphart, A. (1999). Patterns of Democracy.
```

```ts
import { readFileSync, cpSync, mkdirSync } from "node:fs";
import { dirname, join } from "node:path";
import { fileURLToPath } from "node:url";
import yaml from "js-yaml";
import type { SubagentStub } from "../../src/tick.js";
import type { DeskReviewReasonTag } from "../../src/phases/desk_review.js";

const __dirname = dirname(fileURLToPath(import.meta.url));
export const SYNTHETIC_FIXTURES_DIR = join(__dirname, "fixtures");

export type ExpectedOutcome = {
  desk_review: "accept_for_review" | "desk_reject";
  desk_reject_reason_tag?: DeskReviewReasonTag;
  reviewer_recommendations?: Array<"accept" | "accept_with_revisions" | "major_revisions" | "reject">;
  expected_status: "accepted" | "rejected" | "desk_rejected";
  expected_tier?: "unanimous_accept" | "unanimous_reject" | "replication_gate_fail" | "contested";
};

export type ExpectedOutcomes = {
  papers: Record<string, ExpectedOutcome>;
};

export function loadExpectedOutcomes(): ExpectedOutcomes {
  const raw = readFileSync(join(SYNTHETIC_FIXTURES_DIR, "expected-outcomes.yml"), "utf-8");
  return yaml.load(raw, { schema: yaml.JSON_SCHEMA }) as ExpectedOutcomes;
}

/** Copy fixtures/policy and fixtures/papers into a given destination repo root. */
export function seedSyntheticFixture(destinationPublicRepo: string, destinationPolicyRepo: string): void {
  // Policy repo: copy the whole policy/ directory.
  mkdirSync(destinationPolicyRepo, { recursive: true });
  cpSync(join(SYNTHETIC_FIXTURES_DIR, "policy"), destinationPolicyRepo, { recursive: true });

  // Public repo: seed journal + agents + papers.
  mkdirSync(join(destinationPublicRepo, "journals"), { recursive: true });
  mkdirSync(join(destinationPublicRepo, "agents"), { recursive: true });
  mkdirSync(join(destinationPublicRepo, "papers"), { recursive: true });
  cpSync(join(SYNTHETIC_FIXTURES_DIR, "papers"), join(destinationPublicRepo, "papers"), {
    recursive: true,
  });

  // Seed journal inline.
  const journal = [
    "journal_id: agent-polsci-alpha",
    "title: Agent Journal of Political Science",
    "established: 2026-04-17",
    "editor_agent_id: editor-aps-001",
    "scope: |",
    "  Peer-reviewed venue for AI-authored political science research.",
    "submission_fee_cents: 100",
    "status: active",
  ].join("\n");
  require("node:fs").writeFileSync(
    join(destinationPublicRepo, "journals", "agent-polsci-alpha.yml"),
    journal + "\n",
  );

  // Seed author + 3 reviewers.
  const agents = [
    { id: "agent-author01", owner: "user-author" },
    { id: "agent-r1", owner: "user-r1" },
    { id: "agent-r2", owner: "user-r2" },
    { id: "agent-r3", owner: "user-r3" },
  ];
  for (const a of agents) {
    const body = [
      `agent_id: ${a.id}`,
      `owner_user_id: ${a.owner}`,
      `display_name: ${a.id}`,
      `registered_at: "2026-04-01T00:00:00Z"`,
      "topics:",
      "  - comparative-politics",
      "  - electoral-systems",
      "  - replication",
      "  - political-economy",
      "  - institutional-performance",
      "  - voter-turnout",
      "  - misc",
      `review_opt_in: ${a.id === "agent-author01" ? "false" : "true"}`,
      "stats:",
      "  submissions: 0",
      "  acceptances: 0",
      "  reviews_completed: 0",
      "  reviews_timed_out: 0",
      "status: active",
    ].join("\n");
    require("node:fs").writeFileSync(
      join(destinationPublicRepo, "agents", `${a.id}.yml`),
      body + "\n",
    );
  }
}

/** Build a SubagentStub that answers from expected-outcomes.yml for desk_review, throws for others. */
export function buildStub(outcomes: ExpectedOutcomes): SubagentStub {

    },
```

```ts
import yaml from "js-yaml";

export type ReportPaper = {
  paper_id: string;
  expected: string;
  got: string;
  expected_reason?: string;
  got_reason?: string;
  match: boolean;
};

export type ReportResult = {
  passed: number;
  total: number;
  papers: ReportPaper[];
};

type ExpectedOutcome = {
  desk_review: "accept_for_review" | "desk_reject";
  desk_reject_reason_tag?: string;
  expected_status: string;
};

export function runSyntheticReport(
  publicRepoPath: string,
  expectedOutcomesPath: string,
): ReportResult {
  const expectedRaw = readFileSync(expectedOutcomesPath, "utf-8");
  const expected = yaml.load(expectedRaw, { schema: yaml.JSON_SCHEMA }) as {
    papers: Record<string, ExpectedOutcome>;
  };
  const papers: ReportPaper[] = [];
  let passed = 0;
  for (const [paperId, e] of Object.entries(expected.papers)) {
    const metaPath = join(publicRepoPath, "papers", paperId, "metadata.yml");
    const metaRaw = readFileSync(metaPath, "utf-8");
    const gotStatus = metaRaw.match(/^status:\s*(\S+)/m)?.[1] ?? "(missing)";
    const gotReason = metaRaw.match(/^desk_reject_reason_tag:\s*(\S+)/m)?.[1];
    const match =
      gotStatus === e.expected_status &&
      (e.desk_reject_reason_tag ? gotReason === e.desk_reject_reason_tag : true);
    if (match) passed++;
    papers.push({
      paper_id: paperId,
      expected: e.expected_status,
      got: gotStatus,
      expected_reason: e.desk_reject_reason_tag,
      got_reason: gotReason,
      match,
    });
  }
  return { passed, total: papers.length, papers };
}

export function formatReport(result: ReportResult): string {
  const lines: string[] = [];
  for (const p of result.papers) {
    const mark = p.match ? "✓" : "✗";
    const reasonBit = p.expected_reason
      ? ` (reason: ${p.got_reason ?? "(missing)"})`
      : "";
    lines.push(
      `${p.paper_id}: expected=${p.expected} got=${p.got}${reasonBit} ${mark}`,
    );
  }
  lines.push(`${result.passed}/${result.total} passed`);
  return lines.join("\n");
}
```
